Mobile App Usability Testing Report

Usability Analysis – Mobile Banking Application

Overview

This usability testing report focuses on a mobile banking app prototype. The goal was to evaluate ease of navigation, task efficiency, and user satisfaction through real user testing sessions.

Test Objectives

  • Assess how easily users can complete core banking tasks (transfer, check balance, pay bills).
  • Identify usability issues that hinder efficiency or clarity.
  • Measure user satisfaction through post-test interviews.

Test Method

Five participants (ages 21–45) were asked to perform specific tasks within a clickable prototype built in Figma. Screen recording and user observation were used to collect behavioral and emotional feedback.

Key Findings

  1. Navigation Confusion: 60% of users hesitated when locating the “Transfer Funds” option.
  2. Button Size: Several touch targets were too small on mobile screens.
  3. Confirmation Feedback: Lack of clear confirmation messages after transactions caused uncertainty.

Recommendations

  • Reposition the “Transfer Funds” option in the primary navigation bar.
  • Increase button and icon size for better tap accuracy.
  • Add transaction confirmation messages with success icons.
