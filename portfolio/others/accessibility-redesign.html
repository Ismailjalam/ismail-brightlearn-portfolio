Accessibility Redesign – UX Case Study

Improving inclusivity, readability, and interaction for all users

Overview

This project focuses on redesigning a website to meet WCAG 2.1 accessibility standards. The goal was to create an inclusive interface that enhances usability for people with visual, auditory, and motor impairments.

Problem Statement

The previous interface used low-contrast colors, lacked keyboard navigation, and did not include ARIA labels, creating barriers for users with disabilities.

User Research & Insights

  • 5 users with visual impairments tested the original interface.
  • Screen reader navigation failed on several menu items.
  • Keyboard focus was not visible on interactive elements.

Redesign Solutions

✅ Color & Contrast

Used accessible color palettes ensuring contrast ratios above 4.5:1.

✅ Semantic HTML

Replaced <div> heavy layouts with semantic tags like <header>, <nav>, and <main>.

✅ Keyboard Navigation

Added visible focus outlines and improved tab flow for all interactive elements.

✅ ARIA Roles

Implemented ARIA labels for screen readers to describe non-text elements.

✅ Alt Text Audit

Added descriptive alternative text for all images and media.

Before vs After

Before Redesign

Users struggled to read text, find buttons, and navigate forms.

After Redesign

The interface is now screen-reader friendly, fully navigable by keyboard, and visually clear.

Impact & Results

  • Accessibility score improved from 61% → 98% (Lighthouse audit)
  • Average task completion time reduced by 40%
  • User satisfaction increased significantly among visually impaired testers

Lessons Learned

Accessibility is not an afterthought — it’s part of great design. Designing for inclusivity benefits everyone, not just people with disabilities.
