Exercise 23: Accessibility Redesign – UX Compliance

Enhancing inclusivity and usability through accessible design principles.

🎯 Objective

Redesign an existing interface to comply with WCAG 2.1 standards. Focus on visual contrast, keyboard navigation, and screen reader compatibility to make digital products inclusive for all.

🔍 Research & Pain Points

  • Low color contrast causing difficulty for visually impaired users.
  • Missing ARIA labels for icons and non-text content.
  • No focus indicator for keyboard navigation.
  • Non-semantic HTML structure leading to poor screen reader experience.

💡 UX Redesign Approach

The redesign process was guided by four key accessibility principles:

  1. Perceivable: Clear typography, proper contrast, and text alternatives for images.
  2. Operable: Full keyboard navigation and focus visibility.
  3. Understandable: Consistent navigation patterns and predictable layouts.
  4. Robust: Semantic HTML with ARIA support for assistive technologies.

⚙️ Implementation Highlights

Color Contrast

Used accessible color palette ensuring a minimum 4.5:1 contrast ratio.

ARIA Roles

Added descriptive aria-label attributes to non-text elements for screen readers.

Keyboard Flow

Structured navigation order and added visible focus outlines for all interactive components.

Semantic Structure

Replaced generic div tags with header, nav, and main elements.

📊 Outcome & Results

  • Accessibility score improved from 61% → 98% (Lighthouse Audit).
  • Task completion rate increased by 40%.
  • All users could now navigate with keyboard and screen readers efficiently.

🧠 Key Takeaways

Accessibility is not a bonus feature — it’s a core part of great UX design. Every improvement that helps a user with a disability benefits all users overall.
